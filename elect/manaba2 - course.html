質問3


Javaのルールでは:
  • 明示的にコンストラクタを定義しなかった場合，暗黙のうちに引数なしのコンストラクタが定義される．
  • コンストラクタの本体の最初で，親クラスのコンストラクタを明示的に呼び出さなかった場合，暗黙のうちに親クラスタの引数なしのコンストラクタを呼び出すことになる(「super();」と書いたのとおなじになる．


class X {
  X() { System.out.println("[X()]");}               // 行1
  X(int i) { System.out.println("[X(int i)]");}     // 行2
}
class Y extends X {
  Y() { System.out.println("[Y()]"); }              // 行3  まず，親クラスの引数なしのコンストラクタが自動的に呼び出される．  
  Y(int i) { System.out.println("[Y(int i)]"); }    // 行4  ここでもまず，親クラスの引数なしのコンストラクタが自動的に呼び出される．  
}
class Z extends Y {}　  コンストラクタを明示的に定義しないと，引数なしのコンストラクタが自動的に定義される．その中からは，親クラスのコンストラクタを自動的に呼び出す．  
 
上記のようにクラスX,Yが宣言されている場合，次のプログラムコードを実行した結果はどうなると思うか？ 
(選択必須)
Y y0 = new Y();　　
Y y1 = new Y(10);
